11. Pull Request

🔄
Pull Request（PR）是代码审查的核心，也是团队协作的关键流程！

什么是Pull Request？

PR是请求把你的代码合并到主分支的申请，让团队成员可以：

  • 审查你的代码
  • 提出修改建议
  • 讨论实现方案
  • 确保代码质量

创建PR的流程

  1. 在本地创建并切换到新分支
  2. 开发功能并提交代码
  3. 推送到GitHub
  4. 在GitHub上点击"New pull request"
  5. 填写标题和描述，选择要合并的分支
  6. 点击"Create pull request"
# 创建分支并开发
git checkout -b feature-login
# ... 修改代码 ...
git add .
git commit -m "添加登录功能"

# 推送到GitHub
git push -u origin feature-login

代码审查

团队成员可以在PR中：

  • 查看代码变更
  • 添加评论和建议
  • 请求修改或批准
  • 最终合并到主分支

PR最佳实践

1. 每个PR只做一件事
2. 写清楚PR描述
3. 关联相关Issue
4. 及时回复审查意见
