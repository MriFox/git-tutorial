7. 撤销操作

⏪
改错了？没关系！Git提供了多种"后悔药"帮你撤销操作。

撤销工作区修改

# 撤销单个文件的修改
git checkout -- filename.txt

# 撤销所有修改（慎用！）
git checkout -- .

撤销暂存

# 把文件从暂存区移回工作区
git reset HEAD filename.txt

回退提交

# 回退到上一个提交（保留修改在工作区）
git reset --soft HEAD~1

# 回退并撤销修改（不保留修改）
git reset --hard HEAD~1

# 回退到指定提交
git reset --hard commit-id
⚠️ git reset --hard 会丢失修改，操作前请确认！

暂存工作

临时保存当前修改，去做别的事：

# 保存当前修改
git stash

# 查看保存列表
git stash list

# 恢复最近保存
git stash pop

# 删除保存
git stash drop
