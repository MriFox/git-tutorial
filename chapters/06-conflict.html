6. 解决冲突

⚔️
冲突不是错误，而是Git在提醒你："这里需要人工决定保留哪个版本！"

什么是冲突？

当两个人修改了同一个文件的同一行代码，Git无法自动决定保留哪个版本，就会产生冲突。

冲突长什么样？

<<<<<<< HEAD
这是你修改的内容
=======
这是别人修改的内容
>>>>>>> feature-branch

解决步骤

  1. 打开有冲突的文件
  2. 找到冲突标记（<<<<<<<、=======、>>>>>>>）
  3. 决定保留哪个版本，删除冲突标记
  4. 保存文件
  5. 添加到暂存区并提交
# 解决冲突后
git add .
git commit -m "解决冲突：保留最终版本"

预防冲突的技巧

1. 经常pull最新的代码
2. 小步提交，不要积攒太多修改
3. 团队成员分工明确，避免同时修改同一文件
